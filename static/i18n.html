Internationalization

PHPTAL implements the i18n namespace.

This namespace defines the following attributes :

  • i18n:translate
  • i18n:name
  • i18n:attributes

Which allow your templates to be translated *quite* easily (see the phptal manual).

There are two ways of internationalizing PHPTAL templates : the runtime way and the pre-production way.

Runtime

PHPTAL will use gettext and you .po file to translate every template.

It works, it is not so difficult once gettext system is understood but it is slow and it takes CPU/memory/IO on your servers.

project structure example

project/
	src/*.php  # your PHP project sources
	www/       # public pages and calls to you src project
	tpl/       # original not translated templates
	locales/   # .pot and .po files used for internationalization

Pre-production

You can use some of our tools to quickly translate your templates BEFORE sending them to your webserver.

It works, it is simple, it is really fast, but you will have to generate your templates before each upload to your website.

project structure example

project/
	src/*.php  # your PHP project sources
	www/       # public pages and calls to you src project
	tpl/       # original not translated templates
	locales/   # .pot and .po files used for internationalization
	tpl_fr/    # generated using tpl and .fr.po
	tpl_en/    # generated using tpl and .en.po
